# Quickstart — Containerized Database & Cache Runtime

Validates the feature end-to-end. Assumes Docker (Compose v2) installed and a checkout on branch
`008-docker-cache-database`. Run from the repo root.

## 1. Configure

```bash
cp .env.example .env      # fill in local dev credentials (no real secrets committed)
```

## 2. Bring up the full backend stack (one command)

```bash
make up                   # or: docker compose up -d --build
docker compose ps         # all services should reach (healthy)
```

**Expect**: postgres, redis, operations/hr/crm/procurement/workflow services, and gateway all `healthy`.
Gateway is the only container with a published host port (`8081`). (reference-service is a dormant
skeleton and is not started.) (SC-001, SC-005)

## 3. End-to-end through the gateway

```bash
curl -s http://localhost:8081/actuator/health          # {"status":"UP"}
curl -s http://localhost:8081/api/operations/...        # a DB-backed read via the gateway → service → Postgres
```

**Expect**: a correct response backed by the containerized database. Direct DB/Redis access is refused:

```bash
psql postgresql://kita@localhost:5432/kita -c '\dt'     # connection refused (not host-exposed)
```

(SC-003, SC-005)

## 4. Schema isolation

```bash
docker compose exec postgres psql -U kita -d kita -c '\dn'    # schemas: operations, hr, crm, procurement, workflow, public
docker compose exec postgres psql -U kita -d kita -c "select table_schema,count(*) from information_schema.tables group by 1;"
```

**Expect**: each service's tables live only in its own schema; `public` holds only shared extensions.

## 5. Cache behavior (operations-service catalog demonstrator)

- Read a cache-eligible resource twice → second read served from Redis (fewer DB queries). (SC-006)
- Change it, read again → new value (no stale). (C3)
- `docker compose stop redis`, read again → still correct from Postgres. (SC-007, graceful degradation)

## 6. Persistence across restart

```bash
# write some data, then:
docker compose restart postgres        # (or: down && up, without -v)
# re-read → data still present
```

**Expect**: 0 data loss. (SC-002)

## 7. Parity check

```bash
grep -R "postgres:16-alpine\|redis:7.4-alpine" docker-compose.yml   # pinned engine versions
# same currentSchema=<svc>,public and same keys used by the production config
```

**Expect**: identical engine major/minor and config mechanism local↔prod; only values differ. (SC-004, SC-008)

## 8. Teardown

```bash
make down            # stop, KEEP data (docker compose down)
make clean           # clean slate (docker compose down -v) — removes volumes
```

(FR-009)
